Inside the Cub3D DDA Raycaster

What the hell is DDA (Digital Differential Analyzer)?
In raycasting, shooting a ray blindly by tiny steps like 0.01 until it hits a wall is extremely slow and skips corners. DDA is a math trick that calculates exactly where the ray intersects the map's horizontal and vertical grid lines. It lets the ray "jump" cleanly from one cell boundary to the next without missing anything, making it perfectly precise and lightning fast.

To cast a frame, we look at every single pixel column x on the screen. We assign an imaginary coordinate cameraX that spans from -1 (left screen edge) to +1 (right screen edge).

cameraX = 2 * x / screen_width - 1
ray_dir_x = dir_x + plane_x * cameraX
ray_dir_y = dir_y + plane_y * cameraX

Delta Distance is a constant size for a given ray. It answers: If the ray travels perfectly out, how far does it travel to clear exactly 1 unit (1 grid box) in the X axis? What about the Y axis?

The formulas are simple geometry (using Pythagoras):
delta_dist_x = |1 / ray_dir_x|
delta_dist_y = |1 / ray_dir_y|

Side Distance is your starting distance. Since the player isn't perfectly on a grid line, the ray has to travel a custom distance just to hit the very first X-line and Y-line. After it hits the first line, all future bumps use the constant deltaDist.

For example, if ray goes right (ray_dir_x > 0): side_dist_x = (map_x + 1.0 - pos_x) * deltaDistX
If it goes left (ray_dir_x < 0): side_dist_x = (pos_x - map_x) * deltaDistX

The Loop: The ray compares sideDistX and sideDistY.
If sideDistX is smaller, it means the next grid-line we'd hit is an X-line. We jump there! We add deltaDistX to our counter, and move 1 cell in the X axis.
We repeat this, "climbing the ladder" of the grid lines, until we land inside a cell that is a wall (map[y][x] > 0). This logic guarantees we check every single cell the ray touches.

If we draw wall heights using the real Euclidean distance from the player to the wall, we get a rounded "Fisheye" distortion at the edges of our screen (because the edges are actually further away).

To fix it, we use Perpendicular Distance, which projects the hit point straight back perpendicular to the camera plane.
perpWallDist = (side == 0) ? (sideDistX - deltaDistX) : (sideDistY - deltaDistY)
